chap 1: 简介

用途

  • 构建工具（自动化构建，跨平台）
  • 依赖管理
  • 项目信息管理

    chap 2: 安装和配置

  • conf/setting.xml 中全局的定制Maven的行为。

    chap 3: 入门

    编写POM.xml

  • 定义项目的基本信息
  • 定义项目基本的坐标：
    • groupId: 项目属于哪个组
    • artifactId: 当前项目在组中唯一的Id
    • version: 当前项目版本
  • 元素：
    • dependencies,包含多个 dependency元素，声明项目依赖(scope 声明依赖范围[test, compile])
  • 基本命令（相应插件的插件目标，以下命令是按照顺序执行的）：
    • clean: 清理目录 target/
    • compile: 编译项目主代码至target/classes，生成.class文件
    • test: 执行单元测试
    • package: 打包
    • install: 把打包生成的jar安装到本地maven仓库

chap 5: 坐标和依赖

坐标元素

  • groupId：属于哪个项目
  • artifactId：实际项目中的一个Maven项目（模块）
  • version：版本
  • packaging：打包方式，默认jar

依赖：

  • 依赖的配置
    • groupId，artifactId，version: 依赖的坐标配置
    • type：对应坐标定义中的packaging，默认jar
    • scope：依赖的范围
    • optional：依赖是否可选
    • exclusions：排除传递性依赖
  • 依赖范围
  • 传递性依赖

  • 依赖调节

    • mvn dependency:list
    • mvn dependency:tree (依赖树)
    • mvn dependency:analyze （分析依赖）

chap 6: 仓库

  • 统一存储所有Maven 项目共享的构件
  • 分类：

chap 7: 生命周期和插件

生命周期

  • 对所有的构建过程进行抽象和统一，是抽象的，实际任务都是交由插件来完成的。每个生命周期包含一些阶段，阶段是有顺序的，且依赖于前面的阶段。
  • 三套生命周期：clean，default（核心，包含构建的全部步骤），site

命令行

插件

  • 插件目标：mvn 插件:目标 （mvn dependency:tree –dependency插件的tree目标)
  • 插件绑定：生命周期的阶段和插件的目标相互绑定
    • 内置绑定（Maven自身为一些主要的生命周期绑定了插件目标）
    • 自定义绑定（用户定义把某个插件目标绑定到生命周期中的某个阶段）
  • 插件配置：
    • 命令行插件配置： mvn install -Dmaven.test.skip = true
    • pom 中插件全局配置

chap 8: 聚合与继承

  • 聚合（方便快速构建项目）
    • 多个模块聚合在一个模块中一起执行mvn命令

  • 继承（消除重复配置）

    • 父POM中声明一些配置供子POM继承
    • groupId 和 version 可以被继承
    • 元素：不会引入实际的依赖，能约束dependencies下的依赖使用，子模块只需要简单的配置groupId 和 artifactId就能获得对应的依赖信息，引入正确的依赖
    • 类似的 pluginManagement , 管理插件

  • 反应堆

    • 根据聚合模块中声明的顺序来构建
    • Maven还会考虑到模块之间的继承与依赖的关系来决定顺序（依赖关系必须是有向非循环图）
    • 剪裁反应堆：选择构建整个项目或者单个模块

chap 10: 测试

  • mvn cobertura:cobertura 生成测试报告（ target/site/cobertura/index.html ）
  • 持续集成（hudson）

chap 13: 版本管理

  • 版本控制：借助版本控制工具追踪代码的每一个变更
  • 版本管理：项目整体版本的演变过程管理
    • 快照版
    • 发布版
      • 所有自动化测试全部应当通过
      • 项目没有配置任何快照版本的依赖
      • 项目没有配置任何快照版本的插件
      • 所包含的代码已经全部提交到版本控制系统中
